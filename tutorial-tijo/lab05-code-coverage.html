Pokrycie kodu testami

Pokrycie kodu testami (ang. code coverage) to metryka stosowana w testowaniu oprogramowania, która określa, jak wiele linii kodu źródłowego zostało wykrytych (skontrolowanych) przez testy automatyczne.

Implementacja

Umieść w pakiecie produkcyjnym implementację klasy Calculator.

Testy

Umieść w pakiecie testowym implementację klasy CalculatorSpecification.

Testy z raportem

Postaraj się uruchomić testy z raportem pokrycia kodu testami.

Analiza

Po uruchomieniu testów zwróć uwagę na:

  • tabelkę Coverage, która prezentuje pokrycie kodu (klasy, metody, linie) testami;
  • kod klasy Calculator. Kolor czerwony wskazuje linie, które nie są pokryte testami. Kolor zielony wskazuje linie, które są pokryte testami.

Dopisz pozostałe testy i doprowadź do pokrycia 100%

Zadanie

Twoje zadanie będzie polegało na implementacji operacji na bankomacie. Skorzystaj z dostarczonego interfejsu (dokumentacja). Podczas implementacji testów badaj pokrycie kodu testami. 

/**
 * Interfejs reprezentujący bankomat.
 */
public interface ATMInterface {
    
    /**
     * Metoda do sprawdzania salda na koncie.
     * @param pin PIN użytkownika.
     * @return Saldo konta użytkownika.
     * @throws InvalidPinException Jeśli PIN jest nieprawidłowy.
     */
    double checkBalance(int pin) throws InvalidPinException;
    
    /**
     * Metoda do wpłacania środków na konto.
     * @param pin PIN użytkownika.
     * @param amount Kwota do wpłacenia.
     * @return Aktualne saldo konta po dokonaniu wpłaty.
     * @throws InvalidPinException Jeśli PIN jest nieprawidłowy.
     */
    double deposit(int pin, double amount) throws InvalidPinException;
    
    /**
     * Metoda do wypłacania środków z konta.
     * @param pin PIN użytkownika.
     * @param amount Kwota do wypłacenia.
     * @return Aktualne saldo konta po dokonaniu wypłaty.
     * @throws InsufficientFundsException Jeśli saldo jest niewystarczające.
     * @throws InvalidPinException Jeśli PIN jest nieprawidłowy.
     */
    double withdraw(int pin, double amount) throws InsufficientFundsException, InvalidPinException;
}

Czy wiesz, że...

System odporny na awarie to taki, który natychmiast zgłasza na swoim interfejsie każdy stan, który może wskazywać na awarię (technika Fail Fast). Systemy odporne na awarie są zwykle projektowane tak, aby zatrzymać jego normalne działanie, a nie próbować kontynuować potencjalnie wadliwy proces.
