Laboratorium 03: TDD, Git

System zarządzania AT

Twoim celem będzie zaprojektowanie interfejsu dla aplikacji zarządzającej studentami przy użyciu techniki TDD (ang. Test-Driven Development). Aplikacja powinna umożliwiać dodawanie, aktualizowanie, usuwanie studentów, wprowadzanie ocen z przedmiotów oraz obliczanie średniej ocen z przedmiotu. Ponadto, wymagane jest użycie systemu kontroli wersji Git, gdzie po każdym cyklu TDD powinien być wykonany commit do repozytorium zgodnie z ustaloną konwencją.

Wymagania funkcjonalne

Dokumentacja w formie interfejsu

/**
 * Interfejs zarządzania studentami i ocenami.
 * Zawiera metody do dodawania, aktualizowania, usuwania 
 * studentów oraz zarządzania ich ocenami.
 */
public interface StudentManagement {
   
    /**
     * Dodaje nowego studenta do bazy danych.
     * 
     * @param name      Imię studenta.
     * @param surname   Nazwisko studenta.
     * @param studentId Unikalny identyfikator studenta.
     * @return true, jeśli dodanie zakończyło się sukcesem, false w przeciwnym razie.
     */
    boolean addStudent(String name, String surname, String studentId);      
    /**
     * Aktualizuje dane istniejącego studenta na podstawie jego identyfikatora.
     * 
     * @param name      Imię studenta.
     * @param surname   Nazwisko studenta.
     * @param studentId Unikalny identyfikator studenta.
     * @return true, jeśli aktualizacja zakończyła się sukcesem, false w przeciwnym razie.
     */
    boolean updateStudent(String name, String surname, String studentId);       
    /**
     * Usuwa studenta z bazy danych na podstawie jego identyfikatora.
     * 
     * @param studentId Unikalny identyfikator studenta.
     * @return true, jeśli usunięcie zakończyło się sukcesem, false w przeciwnym razie.
     */
    boolean removeStudent(String studentId);        
    /**
     * Dodaje ocenę z danego przedmiotu dla określonego studenta.
     * 
     * @param studentId Unikalny identyfikator studenta.
     * @param subject   Nazwa przedmiotu.
     * @param grade     Ocena.
     * @return true, jeśli dodanie oceny zakończyło się sukcesem, false w przeciwnym razie.
     */
    boolean addGrade(String studentId, String subject, double grade);       
    /**
     * Oblicza średnią ocen z danego przedmiotu dla wszystkich studentów.
     * 
     * @param subject Nazwa przedmiotu.
     * @return Średnia ocen z przedmiotu jako liczba zmiennoprzecinkowa.
     */
    double calculateAverageGrade(String subject);
}

Wymagania dotyczące procesu TDD

Do implementacji powyższych metod należy wykorzystać cykl TDD. Cykl TDD powinien składać się z trzech kroków: Red, Green, Refactor (RGR). Po każdym cyklu RGR należy wykonać commit do repozytorium Git. Commity powinny być opisane zgodnie z ustaloną konwencją.

  • "RGR_1: Implementacja metody odpowiedzialnej za dodanie studenta".
  • "RGR_2: Implementacja metody odpowiedzialnej za aktualizację danych studenta".
  • "RGR_3: Implementacja metody odpowiedzialnej za usunięcie studenta".
  • "RGR_4: Implementacja metody dodawania oceny z przedmiotu".

Czy wiesz, że...

YAGNI (ang. You Aren't Gonna Need It) to zasada w programowaniu, która radzi programistom nie dodawać funkcjonalności, abstrakcji lub optymalizacji do kodu, dopóki nie są one faktycznie potrzebne. Zasada ta kieruje uwagę na aktualne wymagania i unika zbędnej złożoności oraz kosztów związanych z dodawaniem niepotrzebnych funkcji.
